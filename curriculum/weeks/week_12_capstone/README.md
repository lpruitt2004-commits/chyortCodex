# Week 12: Capstone Project & Interview Prep

## Overview
Bring everything together in a full-stack capstone project that showcases your skills. Build a production-ready application from scratch while preparing for technical interviews with algorithms, system design, and behavioral questions.

## Learning Objectives
- Design and implement a complete full-stack application
- Apply best practices from all previous weeks
- Prepare for technical interviews (coding, system design, behavioral)
- Build a portfolio-worthy project
- Practice problem-solving under constraints

## Week Structure

### Days 1-3: Capstone Project
Build a full-stack application with:
- **Frontend**: React with modern hooks and state management
- **Backend**: Node.js/Express REST API with authentication
- **Database**: PostgreSQL with migrations and relationships
- **Testing**: Unit, integration, and E2E tests
- **Deployment**: Dockerized and deployed to cloud
- **Documentation**: README, API docs, architecture diagrams

### Days 4-5: Interview Preparation
- **Algorithms**: Common patterns and data structures
- **System Design**: Scalability, databases, caching
- **Behavioral**: STAR method, common questions
- **Live Practice**: Mock interviews and code challenges

## Capstone Project Options

### Option 1: Task Management System
Full-featured project management app with:
- User authentication and authorization
- Projects, tasks, and subtasks
- Real-time collaboration
- File attachments
- Activity timeline
- Dashboards and analytics

### Option 2: E-Commerce Platform
Online store with:
- Product catalog with search and filters
- Shopping cart and checkout
- User accounts and order history
- Admin dashboard
- Payment integration (Stripe test mode)
- Inventory management

### Option 3: Social Platform
Community-driven application with:
- User profiles and authentication
- Posts, comments, and reactions
- Follow/unfollow functionality
- Real-time notifications
- Image uploads
- Content moderation

### Option 4: Custom Project
Propose your own idea (must include):
- User authentication
- CRUD operations with relationships
- Real-time features or file handling
- Responsive UI
- Deployed to production
